. A user has created a launch configuration for Auto Scaling where CloudWatch detailed monitoring is disabled. The user wants to now enable detailed monitoring. How can the user achieve this?

(A) ** Update the Launch config with CLI to set InstanceMonitoringDisabled = false

(B) The user should change the Auto Scaling group from the AWS console to enable detailed monitoring

(C) Update the Launch config with CLI to set InstanceMonitoring.Enabled = true

(D) Create a new Launch Config with detail monitoring enabled and update the Auto Scaling group

EXPLANATIONS BELOW

Concept note-1: -CloudWatch is used to monitor AWS as well as the custom services. It provides either basic or detailed monitoring for the supported AWS products. In basic monitoring, a service sends data points to CloudWatch every five minutes, while in detailed monitoring a service sends data points to CloudWatch every minute.

Concept note-2: -When you create a launch configuration, you must specify information about the EC2 instances to launch. Include the Amazon Machine Image (AMI), instance type, key pair, security groups, and block device mapping. Alternatively, you can create a launch configuration using attributes from a running EC2 instance.

Concept note-3: -To scale out the AWS resources using manual AutoScaling, which of the below mentioned parameters should the user change? The Manual Scaling as part of Auto Scaling allows the user to change the capacity of Auto Scaling group. The user can add / remove EC2 instances on the fly.

Concept note-4: -On the navigation bar at the top of the screen, choose the same AWS Region that you used when you created the launch configuration. Choose Create an Auto Scaling group. More items
