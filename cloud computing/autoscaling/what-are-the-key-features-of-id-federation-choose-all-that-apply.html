What are the key features of ID Federation? (Choose all that apply)

(A) ** allows others to access an AWS account without having to create IAM Users

(B) grants an AWS account the ability to store an external identity’s credentials

(C) ** provides identities from on-premises systems a way to access an AWS account

(D) ** provides existing identity providers the freedom to manage their identities while giving access to an AWS account

EXPLANATIONS BELOW

Concept note-1: -Federated identity is based on a combination of several components including authentication, authorization, access control, IdPs, and service providers.

Concept note-2: -Federated identity allows authorized users to access multiple applications and domains using a single set of credentials. It links a user’s identity across multiple identity management systems so they can access different applications securely and efficiently.

Concept note-3: -AWS Identity and Access Management (IAM) is a web service that helps you securely control access to AWS resources. With IAM, you can centrally manage permissions that control which AWS resources users can access. You use IAM to control who is authenticated (signed in) and authorized (has permissions) to use resources.

Concept note-4: -Access keys are long-term credentials for an IAM user or the AWS account root user. You can use access keys to sign programmatic requests to the AWS CLI or AWS API (directly or using the AWS SDK). For more information, see Signing AWS API Requests in the Amazon Web Services General Reference.
