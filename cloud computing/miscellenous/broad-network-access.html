Broad network access

(A) ** Cloud computing resources are accessible over the network

(B) Resource pooling

(C) Measured service

(D) Self-service

EXPLANATIONS BELOW

Concept note-1: -Broad network access means that the cloud customer or tenant, whoeverâ€™s consuming a service, does not manage the IT service infrastructure. And as a result, this means that that IT service runs in a provider data center somewhere and is therefore accessible to the consumer over a network.

Concept note-2: -Broad network access Network bandwidth and latency are very important aspects of cloud computing and broad network access, because they relate to the quality of service (QoS) on the network.

Concept note-3: -Static environment is not an essential characteristics of cloud computing. Explanation: Cloud computing is a convenient configurable computational resources.

Concept note-4: -Broad network access: It is critical that the cloud infrastructure be accessible via variety of network-capable devices, e.g. SmartPhones, PCs, laptops, tablets, and so on. Resource pooling: The provider uses an infrastructure which allows for the pooling of physical computing resources to service many consumers.

Concept note-5: -Cloud computing is on-demand access, via the internet, to computing resources-applications, servers (physical servers and virtual servers), data storage, development tools, networking capabilities, and more-hosted at a remote data center managed by a cloud services provider (or CSP).
