Which of the following is not a characteristic of an IAM role?

(A) It can be used only when there is a separate policy called Trusted Policy.

(B) It can also be used in applications or AWS services.

(C) ** A Token is issued along with a fixedly used Access Key and Secret Key.

(D) If necessary, it is possible to switch to another role according to the authority.

EXPLANATIONS BELOW

Concept note-1: -Here are some important characteristics of user groups: A user group can contain many users, and a user can belong to multiple user groups. User groups can’t be nested; they can contain only users, not other user groups. There is no default user group that automatically includes all users in the AWS account.

Concept note-2: -What best describes an IAM role? A role is something that a user, application or service can “assume” to receive temporary security credentials that provide access to a resource.

Concept note-3: -An IAM user is an entity that represents a person or service. Can be assigned: An access key ID and secret access key for programmatic access to the AWS API, CLI, SDK, and other development tools.

Concept note-4: -Shared access to the AWS account. The main feature of IAM is that it allows you to create separate usernames and passwords for individual users or resources and delegate access. Granular permissions. Multifactor authentication (MFA). Identity Federation. Free to use. PCI DSS compliance. Password policy. 23-Mar-2023
