AXIS framework is provided by which company

(A) IBM

(B) ** Apache Foundation

(C) Redhat

(D) JBOSS

EXPLANATIONS BELOW

Concept note-1: -Today, Apache Axis is to a large extend superseded by a new generation of SOAP stacks such as Apache Axis2, Apache CXF and Metro. However, Axis is still relevant for the following type of projects: Projects that need to use JAX-RPC.

Concept note-2: -The current release is 1.8. 2 and was published on July 14, 2022. The release note for this release can be found here. The binary distribution contains all the Axis2 libraries and modules, except for Apache Rampart (WS-Security implementation) which must be downloaded separately.

Concept note-3: -The main differences between axis2 web service and CXF web service are as follows: CXF has support for WS-Addressing, WS-Policy, WS-RM, WS-Security, and WS-I BasicProfile. Axis2 supports each of these except for WS-Policy, which will be supported in an upcoming version.

Concept note-4: -Apache Axis2 is a web service engine. It is a redesign and re-write of the widely used Apache Axis SOAP stack. Implementations of Axis2 are available in Java and C. Apache Axis2.
