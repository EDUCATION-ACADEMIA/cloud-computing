REST is a type of web service protocol.

(A) True

(B) ** False

EXPLANATIONS BELOW

Concept note-1: -REST is web standards based architecture and uses HTTP Protocol. It revolves around resource where every component is a resource and a resource is accessed by a common interface using HTTP standard methods. REST was first introduced by Roy Fielding in 2000.

Concept note-2: -The architectural style is: although the high-level architectural design of any work is given, the low-level design is kept free for the implementing party (vendor, provider). REST is not a standard. Although REST is not a standard, it is implemented through web standards in modern days. REST is not a protocol.

Concept note-3: -In the web services world, REpresentational State Transfer (REST) is a key design idiom that embraces a stateless client-server architecture in which the web services are viewed as resources and can be identified by their URLs.

Concept note-4: -REST is an API architecture style. It stands for representational state transfer. REST specifies how data is presented to a client in a format that is convenient for the client.

Concept note-5: -A RESTful API uses existing HTTP methodologies defined by the RFC 2616 protocol, such as: GET to retrieve a resource; PUT to change the state of or update a resource, which can be an object, file or block; POST to create that resource; and.
